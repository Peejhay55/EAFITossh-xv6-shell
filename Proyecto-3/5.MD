# Documentación Técnica: Simulación de Memory Mapped Files (mapzero)

En esta etapa final se utilizó la base de *Lazy Allocation* para simular el comportamiento de `mmap`, permitiendo reservar regiones de memoria que se inicializan bajo demanda con un patrón específico.

## 1. Definición de la Región Virtual

Se introdujo la capacidad de que cada proceso gestione una región especial de memoria simulada.

### Cambios en Estructuras:
- **`struct vregion` ([kernel/proc.h](kernel/proc.h))**: Nueva estructura que almacena la dirección de inicio (`start`) y el tamaño (`size`) de la región mapeada.
- **`struct proc`**: Se añadió un campo `vreg` de tipo `struct vregion`.

## 2. Nueva System Call: `mapzero(size)`

Se implementó la syscall `mapzero` para permitir que los programas de usuario reserven este espacio.

### Comportamiento de `sys_mapzero()` ([kernel/sysproc.c](kernel/sysproc.c)):
1. Calcula una dirección de inicio alineada a página después del actual `sz`.
2. Registra los límites en `p->vreg`.
3. Incrementa `p->sz` para que el rango sea considerado legal por el kernel.
4. **No asigna memoria física**, siguiendo la filosofía *Lazy*.

## 3. Manejo de Fallos Especializado

Se extendió la lógica de `usertrap()` en [kernel/trap.c](kernel/trap.c) para dar un tratamiento especial a esta región.

### Lógica en `usertrap()`:
- Al ocurrir un Page Fault, si el kernel detecta que la dirección (`stval`) pertenece a la región definida en `p->vreg`:
  - Asigna la página físicamente (vía `vmfault`).
  - **Diferenciador**: Inicializa toda la página con el carácter `'A'` usando `memset`.
  - Esto simula la carga de un archivo o una inicialización específica del sistema.

## 4. Validación con `tmmap_sim`

El programa [user/tmmap_sim.c](user/tmmap_sim.c) demuestra el flujo completo:
1. Llama a `mapzero()`.
2. Accede a la memoria recién "mapeada".
3. Verifica que, aunque el programa nunca escribió nada, la memoria ya contiene el patrón `'A'` gracias a la intervención del kernel durante el Page Fault.
